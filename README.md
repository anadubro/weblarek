# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
### Интерфейсы
##### Интерфейс IProduct
Объект товара, содержащий информацию о его идентификаторе, названии, изображении, категории, цене и описании. Используется для отображения каталога, карточек товара и формирования содержимого заказа.

Поля интерфейса:
`id: string` - идентификатор товара.
`title: string` - название товара.
`image: string` - изображение товара.
`category: string` - наименование категории товара.
`price: number | null` - цена за товар.
`description: string` - описание товара.

##### Интерфейс  IBuyer 
Объект покупателя, включающий данные о способе оплаты, адресе, электронной почте и телефоне. Необходим для оформления заказа, передачи контактных данных и выбора способа оплаты.

Поля интерфейса:
`payment: TPayment` - способ оплаты с выбором из нескольких значений, определяемых типом `TPayment = 'card' | 'cash' | 'online'`.
`address: string` - адрес покупателя.
`email: string` - электронный адрес покупателя.
`phone: string` - номер телефона покупателя.

##### Интерфейс IBuyerErrors
Объект с информацией о найденных ошибках в процессе валидации полей о покупалях.
Все поля интерфейса являются необязательными.

Поля интерфейса:
`payment?: string` - сообщение об ошибке способа оплаты.
`address?: string` - сообщение об ошибке в адресе покупателя.
`email?: string` - сообщение об ошибке в электронной почте покупателя.
`phone?: string` - сообщение об ошибке в номере телефона покупателя.

#### Модели данных
##### Модель Catalog
Отвечает за хранение списка всех товаров и выбранного товара для подробного описания.
Работает с интерфесом IProduct.

Конструктор:
`constructor()` - пока пуст.

Поля:
`productList: IProduct[]` - массив объектов товаров. Хранит актуальный список товаров для отображения в каталоге.
`selectedProduct: IProduct | null` - выбранный пользователем товар. Используется для открытия карточки или передачи данных другим компонентам.

Методы:
`setProducts(products: IProduct[]): void` - сохраняет список товаров каталога, на вход принимает и список товаров.
`getProducts(): IProduct[]` - возвращает список товаров каталога.
`setSelectedProduct(product: IProduct): void `- сохраняет выбранный пользователем товар, на вход принимает объект товара.
`getSelectedProduct(): IProduct | null` - возвращает выбранную карточку.
`getProductById(productId: string): IProduct` - возвращает объект товара по id.

##### Модель Cart
Отвечает за хранение товаров, добавленных в корзину.
Работает с интерфесом IProduct.

Конструктор:
`constructor()`

Поля:
`products: IProduct[]` - хранит массив товаров, выбранных покупателем для покупки.

Методы:
`getProducts(): IProduct[]` - возвращает массив товаров в корзине.
`addProduct(product: IProduct): void` - добавляет товар в корзину.
`removeProduct(product: IProduct): void` - удаляет товар из корзины.
`clear(): void` - отчистка корзины.
`getTotalPrice(): number` - общая сумма товаров в корзине.
`getCount(): number` - количество товаров в корзине.
`hasProductById(productId: string): boolean` - проверяет наличие товара в корзине по id.

##### Модель Buyer
Отвечает за хранение информации о покупателе.
Работает с интерфесом IBuyer.

Конструктор:
` constructor(buyer?: IBuyer | undefined)` - это необязательный аргумент. Если задан, принимает на вход объект с информацией о покупателе и сохраняет ее.

Поля:
`payment: TPayment` - способ оплаты с выбором из нескольких значений, определяемых типом `TPayment = 'card' | 'cash' | 'online'`.
`address: string` - адрес покупателя.
`email: string` - электронный адрес покупателя.
`phone: string` - номер телефона покупателя.

Методы:
`setPayment(payment: TPayment): void` - сохраняет данные о методе оплаты.
`setAddress(address: string): void` - сохраняет данные об адресе покупателя.
`setEmail(email: string): void` - сохраняет данные о электронной почте покупателя.
`setPhone(phone: string): void` - сохраняет данные о номере телефона покупателя.
`getData(): IBuyer` - получает все данные о покупателе.
`clear(): void` - отчищает все данные о покупателе.
`validate(): object` - валидация полей ввода, возвращает объект с указанием ошибки для каждого поля.

### Слой коммуникации
#### Типы
##### Тип TProductListInfo
Тип описывает данные о товарах и их количество, получаемые с сервера.

Поля типа:
`total: number` - общее количество товаров.
`items: IProduct` - массив с данными о товарах.

##### Тип TOrderInfo
Тип описывает данные о заказе и о покупателе, передаваемые на сервер.

Поля типа:
`payment: TPayment` - способ оплаты с выбором из нескольких значений, определяемых типом `TPayment = 'card' | 'cash' | 'online'`.
`address: string` - адрес покупателя.
`email: string` - электронный адрес покупателя.
`phone: string` - номер телефона покупателя.
`total: number` - общая стоимость корзины.
`items: string[]` - массив id добавленных в корзину товаров.

#### Класс Communication
Отвечается за коммуникацию с сервером: получает список товаров с сервера, отправляет информацию о заказе на сервер.

`constructor(api: IApi)` - принимает и сохраняет объект, соответствующий интерфейсу IApi.

Поля класса:
`api: IApi` - объект, соответствующий интерфейсу IApi.

Методы класса:
`getProductList(): : Promise<IProduct[]>` - получает список товаров.
`postOrderInfo(buyer: IBuyer, cart: Cart): void` - принимает объект покупателя и объект корзины, и отправляет на сервер информацию о заказе(информацию о покупателе и список выбранных товаров).


### Слой View
#### Интерфейсы и типы слоя View
##### Интерфейс IHeaderView
Интерфейс описывает данные, необходимые для отображения счетчика корзины в шапке сайта. Используется классом HeaderView.

Поля интерфейса:
`counter: number` - количество товаров, находящихся в корзине.

##### Интерфейс IModalView
Интерфейс описывает данные, необходимые для отображения содержимого модального окна.

Поля интерфейса:
`content: HTMLElement` - DOM-элемент, который будет отображён внутри модального окна.

##### Интерфейс IProductBase
Базовый интерфейс для карточек товара. Используется в базовом классе карточек и наследуется другими интерфейсами.

Поля интерфейса:
`title: string` - название товара.
`price: number | null` - цена товара. Если значение null, товар считается бесценным.

##### Тип TPrice
Тип описывает цену товара. Используется в базовом классе карточек товара ProductBaseView и всех его наследниках.

Тип:
`TPrice = number | null`

Описание:
`number` - цена товара в синапсах
`null` - товар не имеет цены и считается бесценным

##### Интерфейс ICatalogProductItemView
Интерфейс карточки товара, отображаемой в каталоге на главной странице. Наследуется от IProductBase.

Поля интерфейса:
`title: string` - название товара.
`price: number | null` - цена товара.
`category: string` - категория товара.
`image: string` - путь к изображению товара.

##### Интерфейс IProductItemActions
Интерфейс описывает набор действий пользователя для карточки товара в каталоге. Используется в классе CatalogProductItemView.
Используется для передачи обработчиков действий из Презентера в компонент представления.

Поля интерфейса:
`onClickOpen: () => void` - обработчик клика по карточке товара, вызывается при выборе товара для просмотра подробной информации.

##### Интерфейс IProductView
Интерфейс карточки товара с подробным описанием. Используется в модальном окне просмотра детальной информации о товаре. Наследуется от IProductBase.

Поля интерфейса:
`title: string` - название товара.
`price: number | null` - цена товара.
`category: TCategoryKey` - категория товара.
`image: string` - путь к изображению товара.
`description: string` - описание товара.
`buttonText: string` - текст кнопки.
`isDisabled: boolean` - флаг, определяющий доступность кнопки.

##### Интерфейс IProductActions
Интерфейс описывает действия пользователя в карточке товара с подробным описанием. Используется в классе ProductView.
Позволяет передавать обработчик пользовательского действия в компонент представления и уведомлять Презентер о намерении пользователя.

Поля интерфейса:
`onClick: () => void` - обработчик клика по кнопке действия товара (покупка или удаление из корзины).

##### Интерфейс ICatalogView
Интерфейс для отображения каталога товаров.

Поля интерфейса:
`catalog: HTMLElement[]` - массив DOM-элементов карточек товаров.

##### Интерфейс ICartView
Интерфейс описывает данные, необходимые для отображения содержимого корзины товаров.

Поля интерфейса:
`productList: HTMLElement[]` - массив DOM-элементов товаров, добавленных в корзину.
`cartPrice: number` - общая стоимость товаров в корзине.
`isOrderDisabled: boolean` - флаг, определяющий доступность кнопки оформления заказа.

##### Интерфейс ICartProductItemView
Интерфейс описывает данные для отображения одного товара в корзине. Наследуется от IProductBase.

Поля интерфейса:
`title: string` - название товара.
`price: number | null` - цена товара.
`index: number` - порядковый номер товара в списке корзины.

##### Интерфейс ICartProductItemActions
Интерфейс описывает действия пользователя для товара в корзине.
Используется для передачи обработчика из Презентера в компонент представления.

Поля интерфейса:
`onClickDelete: () => void` - обработчик клика по кнопке удаления товара из корзины.

##### Интерфейс IOrderSuccess
Интерфейс описывает данные, отображаемые в представлении успешного оформления заказа. Используется в классе OrderSuccessView.

Поля интерфейса:
`orderPrice: number` - итоговая стоимость заказа в синапсах.

##### Интерфейс IBaseFormView
Базовый интерфейс состояния формы оформления заказа. Используется как базовый интерфейс для всех форм оформления заказа.

Поля интерфейса:
`isSubmitDisabled: boolean` - управляет доступностью кнопки отправки формы;
`errors: IBuyerErrors` - объект ошибок валидации формы.

##### Интерфейс IBuyerFormData
Интерфейс описывает данные, вводимые пользователем при оформлении заказа. Используется для передачи частичных данных формы через событие orderOrContactsForm:changed.

`payment?: string` - выбранный способ оплаты;
`address?: string` - адрес доставки;
`email?: string` - адрес электронной почты;
`phone?: string` - номер телефона.

##### Интерфейс IOrderFormView
Интерфейс состояния формы оформления заказа. Наследуется от IBaseFormView

Поля интерфейса:
`payment: TPayment` - выбранный способ оплаты;
`address: string` - адрес доставки.

##### Интерфейс IContactsFormView
Интерфейс состояния формы контактных данных. Наследуется от IBaseFormView

Поля интерфейса:
`email: string` - электронная почта покупателя;
`phone: string` - номер телефона покупателя.

##### Класс ProductBaseView
Базовый класс для всех карточек товаров. Содержит общую логику отображения названия и цены товара.
Класс является дженериком и работает с типами, наследующими интерфейс IProductBase.

Конструктор:
`constructor(container: HTMLElement)` - принимает DOM-элемент карточки.

#### События
##### События actions
`cart:open` - Клик по кнопке корзины в шапке сайта. Источник: HeaderView.
`cartView:submit` - Клик по кнопке подтверждения корзины. Источник: CartView.
`orderOrContactsForm:changed` - Генерируется при изменении любого поля форм оформления заказа или контактов. Источники: BaseFormView, OrderFormView, ContactsFormView.
`orderForm:submit` - Отправка первой формы оформления заказа. Источник: OrderFormView.
`contactsForm:submit` - Отправка формы контактных данных. Источник: ContactsFormView.
`productView:selected` - Клик по карточке товара в каталоге. Источник: CatalogProductItemView. Функция-источник: onClickOpen.
`productView:toggle` - добавляет или удаляет товар в модель корзины по клику на кнопке 'купить/удалить' выбранного товара. Источник: ProductView. Функция-источник: onClick.
`cartView:removeProduct` - Удаление товара из корзины: из карточки товара, из списка корзины. Источник: ProductView, CartProductItemView. Функции-источники: onClick, onClickDelete.

##### События слоя Model
`catalogModel:changed` - Изменение списка товаров каталога. Источник: модель Catalog.
`productModel:selected` - Изменение выбранного товара. Источник: модель Catalog.
`cartModel:changed` - Изменение корзины: добавление товара, удаление товара, очистка корзины. Источник: модель Cart.
`buyerModel:changed` - Валидация данных. Источник: модель Buyer.

#### Слой Presenter
Слой Presenter реализован в файле main.ts и выполняет роль связующего звена между слоями Model и View.

##### Presenter: 
1. Создаёт экземпляр брокера событий EventEmitter.
2. Инициализирует модели Catalog, Cart, Buyer.
3. Инициализирует представления (HeaderView, CatalogView, CartView, формы заказа, модальные окна).
4. Загружает список товаров с сервера и передаёт его в модель каталога.

##### Отображение каталога товаров
1. Presenter загружает список товаров с сервера и вызывает catalogModel.setProducts.
2. Модель Catalog генерирует событие catalogModel:changed.
3. Presenter формирует карточки товаров (CatalogProductItemView) и передаёт их в CatalogView.

##### Просмотр детальной карточки товара
1. Пользователь кликает по карточке товара в каталоге.
2. Представление генерирует событие productView:selected.
3. Presenter устанавливает выбранный товар в модель Catalog.
4. Модель генерирует событие productModel:selected.
5. Presenter создаёт ProductView, настраивает кнопку действия (в корзину / удаление из корзины / недоступно) и открывает модальное окно с карточкой товара.

##### Работа с корзиной
Добавление товара:
1. ProductView генерирует событие productView:toggle;
2. Presenter добавляет товар в модель Cart.

Удаление товара:
1. ProductView или CartProductItemView генерирует событие cartView:removeProduct;
2. Presenter удаляет товар из модели Cart.

Обновление интерфейса:
1. модель Cart генерирует событие cartModel:changed;
2. Presenter обновляет:
- список товаров в CartView;
- счётчик товаров в HeaderView.

##### Открытие корзины
1. HeaderView генерирует событие cart:open.
2. Presenter рендерит содержимое корзины (CartView) в модальном окне и открывает его.

##### Оформление заказа
Подтверждение корзины:
1. CartView генерирует событие cartView:submit;
2. Presenter открывает форму оформления заказа (OrderFormView).

Заполнение данных:
1. формы заказа и контактов генерируют событие orderOrContactsForm:changed;
2. Presenter сохраняет введённые данные в модель Buyer;
3. модель генерирует buyerModel:changed, после чего Presenter обновляет состояния форм и отображает ошибки.

Переход между формами:
1. orderForm:submit — открытие формы контактных данных;
2. contactsForm:submit — финальная отправка заказа.

##### Отправка заказ
1. Presenter формирует объект заказа на основе данных модели Buyer и модели Cart.
2. Выполняет отправку заказа на сервер через Communication.
3. При успешном ответе:
- открывает OrderSuccessView;
- очищает модели Buyer и Cart.
4. При ошибке отправки выводит сообщение об ошибке в консоль.

##### Завершение оформления заказа
1. OrderSuccessView генерирует событие orderSuccessView:close;
2. Presenter закрывает модальное окно.

##### События Presenter

`productView:selected` — клик по карточке товара в каталоге. Presenter устанавливает выбранный товар в модель Catalog.
`productModel:selected` — изменение выбранного товара в модели каталога. Presenter открывает модальное окно с детальной карточкой товара (ProductView).
`productView:toggle` — добавляет или удаляет товар в модель корзины по клику на кнопке 'купить/удалить' выбранного товара.
`cartView:removeProduct` — клик по кнопке удаления товара (в карточке товара или в корзине). Presenter удаляет товар из модели Cart.
`cartModel:changed` — изменение состояния корзины. Presenter обновляет содержимое CartView и счётчик товаров в HeaderView.
`cart:open` — клик по значку «Открыть корзину» в компоненте HeaderView. Presenter открывает корзину в модальном окне.
