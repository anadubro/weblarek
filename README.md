# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

### Данные
### Интерфейсы
##### Интерфейс IProduct
Объект товара, содержащий информацию о его идентификаторе, названии, изображении, категории, цене и описании. Используется для отображения каталога, карточек товара и формирования содержимого заказа.

Поля интерфейса:
`id: string` - идентификатор товара.
`title: string` - название товара.
`image: string` - изображение товара.
`category: string` - наименование категории товара.
`price: number | null` - цена за товар.
`description: string` - описание товара.

##### Интерфейс  IBuyer 
Объект покупателя, включающий данные о способе оплаты, адресе, электронной почте и телефоне. Необходим для оформления заказа, передачи контактных данных и выбора способа оплаты.

Поля интерфейса:
`payment: TPayment` - способ оплаты с выбором из нескольких значений, определяемых типом `TPayment = 'card' | 'cash' | 'online'`.
`address: string` - адрес покупателя.
`email: string` - электронный адрес покупателя.
`phone: string` - номер телефона покупателя.

##### Интерфейс IBuyerErrors
Объект с информацией о найденных ошибках в процессе валидации полей о покупалях.
Все поля интерфейса являются необязательными.

Поля интерфейса:
`payment?: string` - сообщение об ошибке способа оплаты.
`address?: string` - сообщение об ошибке в адресе покупателя.
`email?: string` - сообщение об ошибке в электронной почте покупателя.
`phone?: string` - сообщение об ошибке в номере телефона покупателя.

#### Модели данных
##### Модель Catalog
Отвечает за хранение списка всех товаров и выбранного товара для подробного описания.
Работает с интерфесом IProduct.

Конструктор:
`constructor()` - пока пуст.

Поля:
`productList: IProduct[]` - массив объектов товаров. Хранит актуальный список товаров для отображения в каталоге.
`selectedProduct: IProduct | null` - выбранный пользователем товар. Используется для открытия карточки или передачи данных другим компонентам.

Методы:
`setProducts(products: IProduct[]): void` - сохраняет список товаров каталога, на вход принимает и список товаров.
`getProducts(): IProduct[]` - возвращает список товаров каталога.
`setSelectedProduct(product: IProduct): void `- сохраняет выбранный пользователем товар, на вход принимает объект товара.
`getSelectedProduct(): IProduct | null` - возвращает выбранную карточку.
`getProductById(productId: string): IProduct` - возвращает объект товара по id.

##### Модель Cart
Отвечает за хранение товаров, добавленных в корзину.
Работает с интерфесом IProduct.

Конструктор:
`constructor()`

Поля:
`products: IProduct[]` - хранит массив товаров, выбранных покупателем для покупки.

Методы:
`getProducts(): IProduct[]` - возвращает массив товаров в корзине.
`addProduct(product: IProduct): void` - добавляет товар в корзину.
`removeProduct(product: IProduct): void` - удаляет товар из корзины.
`clear(): void` - отчистка корзины.
`getTotalPrice(): number` - общая сумма товаров в корзине.
`getCount(): number` - количество товаров в корзине.
`hasProductById(productId: string): boolean` - проверяет наличие товара в корзине по id.

##### Модель Buyer
Отвечает за хранение информации о покупателе.
Работает с интерфесом IBuyer.

Конструктор:
` constructor(buyer?: IBuyer | undefined)` - это необязательный аргумент. Если задан, принимает на вход объект с информацией о покупателе и сохраняет ее.

Поля:
`payment: TPayment` - способ оплаты с выбором из нескольких значений, определяемых типом `TPayment = 'card' | 'cash' | 'online'`.
`address: string` - адрес покупателя.
`email: string` - электронный адрес покупателя.
`phone: string` - номер телефона покупателя.

Методы:
`setPayment(payment: TPayment): void` - сохраняет данные о методе оплаты.
`setAddress(address: string): void` - сохраняет данные об адресе покупателя.
`setEmail(email: string): void` - сохраняет данные о электронной почте покупателя.
`setPhone(phone: string): void` - сохраняет данные о номере телефона покупателя.
`getData(): IBuyer` - получает все данные о покупателе.
`clear(): void` - отчищает все данные о покупателе.
`validate(): object` - валидация полей ввода, возвращает объект с указанием ошибки для каждого поля.

### Слой коммуникации
#### Типы
##### Тип TProductListInfo
Тип описывает данные о товарах и их количество, получаемые с сервера.

Поля типа:
`total: number` - общее количество товаров.
`items: IProduct` - массив с данными о товарах.

##### Тип TOrderInfo
Тип описывает данные о заказе и о покупателе, передаваемые на сервер.

Поля типа:
`payment: TPayment` - способ оплаты с выбором из нескольких значений, определяемых типом `TPayment = 'card' | 'cash' | 'online'`.
`address: string` - адрес покупателя.
`email: string` - электронный адрес покупателя.
`phone: string` - номер телефона покупателя.
`total: number` - общая стоимость корзины.
`items: string[]` - массив id добавленных в корзину товаров.

#### Класс Communication
Отвечается за коммуникацию с сервером: получает список товаров с сервера, отправляет информацию о заказе на сервер.

`constructor(api: IApi)` - принимает и сохраняет объект, соответствующий интерфейсу IApi.

Поля класса:
`api: IApi` - объект, соответствующий интерфейсу IApi.

Методы класса:
`getProductList(): : Promise<IProduct[]>` - получает список товаров.
`postOrderInfo(buyer: IBuyer, cart: Cart): void` - принимает объект покупателя и объект корзины, и отправляет на сервер информацию о заказе(информацию о покупателе и список выбранных товаров).
